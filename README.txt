Prayas One Student App - Split Files Exporter (GitHub Safe Plain-Text Edition)
========================================================================

Your database has grown large! This split package prevents file upload block errors, security alerts, and size limits on GitHub by exporting the dataset in plain-text (.txt) fragments rather than executable Javascript (.js) scripts.

FILES LIST IN THIS BUNDLE:
1. index.html                   - Main student testing portal.
2. 404.html                     - SPA clean-path router fallback redirect (prayasone.in/Blackbook/A-Word-Test support!).
3. sitemap.xml                  - Automatically compiled XML Sitemap listing all deep URLs for high SEO rank!
4. robots.txt                   - Robots file explicitly linking the Sitemap location to Google crawler bots.
5. config_part_1.txt, _2.txt... - Plain-text database fragments (pure base64 chunks, each under 5MB).
6. README.txt                   - This instruction file.

HOW TO DEPLOY TO GITHUB:
1. Extract/Unzip all the files in this ZIP archive.
2. Upload/Commit all files (index.html, 404.html, sitemap.xml, robots.txt, and all 'config_part_*.txt' files) directly into your GitHub repository root.
3. Make sure all the 'config_part_*.txt' files, '404.html', 'robots.txt' and 'sitemap.xml' are placed in the SAME FOLDER/DIRECTORY as index.html.
4. Enable GitHub Pages in your Repository Settings pointing to this folder or branch.

HOW TO USE CLEAN DYNAMIC TEST URLs (prayasone.in/Blackbook/A-Word-Test):
Since GitHub Pages is a static site host, direct refreshes on routes like /Blackbook/A-Word-Test normally throw a 404. 
But thanks to the included '404.html' file, it automatically redirects requests back to index.html with query parameters, which then reinstates the pristine URL in the student's address bar without any error! 
Just make sure '404.html' is uploaded next to 'index.html'!

GOOGLE SITE RANKING & SITEMAP:
The 'sitemap.xml' dynamically indexes all your exams, categories, subcategories, and topics.
When uploaded, submit 'https://prayasone.in/sitemap.xml' in Google Search Console to successfully crawl and index all your testing pages!

The application dynamically fetches and reassembles all segments into memory upon startup via secure browser fetches!
